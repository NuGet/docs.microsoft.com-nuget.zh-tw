## <a name="configuring-the-packages-folder"></a><span data-ttu-id="1c836-139">設定 Packages 資料夾</span><span class="sxs-lookup"><span data-stu-id="1c836-139">Configuring the Packages folder</span></span>

<span data-ttu-id="1c836-140">使用 `NuGet.Server` 1.5 及更新版本，您可以更明確地使用 `web.config` 的 `appSetting/packagesPath` 值來設定套件資料夾：</span><span class="sxs-lookup"><span data-stu-id="1c836-140">With `NuGet.Server` 1.5 and later, you can more specifically configure the package folder using the `appSetting/packagesPath` value in `web.config`:</span></span>

    <add key="packagesPath" value="C:\MyPackages" />

<span data-ttu-id="1c836-141">`packagesPath` 可以是絕對或虛擬路徑。</span><span class="sxs-lookup"><span data-stu-id="1c836-141">`packagesPath` can be an absolute or virtual path.</span></span>

<span data-ttu-id="1c836-148">如果您的伺服器已受保護或本就不需要 API 金鑰 (例如，在本機小組網路上使用私用伺服器)，您可以將 `requireApiKey` 設成 `false`。</span><span class="sxs-lookup"><span data-stu-id="1c836-148">If your server is already secured or you do not otherwise require an API key (for example, when using a private server on a local team network), you can set `requireApiKey` to `false`.</span></span> <span data-ttu-id="1c836-149">接著，所有能夠存取伺服器的使用者就都可以推送套件。</span><span class="sxs-lookup"><span data-stu-id="1c836-149">All users with access to the server can then push packages.</span></span>
