## <a name="cursor"></a><span data-ttu-id="ff249-443">資料指標</span><span class="sxs-lookup"><span data-stu-id="ff249-443">Cursor</span></span>

### <a name="overview"></a><span data-ttu-id="ff249-444">概觀</span><span class="sxs-lookup"><span data-stu-id="ff249-444">Overview</span></span>

<span data-ttu-id="ff249-445">本節說明的用戶端概念（雖然不一定是由通訊協定強制執行）應該屬於任何實際的目錄用戶端。</span><span class="sxs-lookup"><span data-stu-id="ff249-445">This section describes a client concept that, although is not necessarily mandated by the protocol, should be part of any practical catalog client implementation.</span></span>

<span data-ttu-id="ff249-446">因為目錄是依時間編制索引的附加資料結構，所以用戶端應該在本機儲存**游標**，表示用戶端處理目錄專案的時間點。</span><span class="sxs-lookup"><span data-stu-id="ff249-446">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="ff249-447">請注意，此資料指標的值絕不能使用用戶端的電腦時鐘來產生。</span><span class="sxs-lookup"><span data-stu-id="ff249-447">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="ff249-448">相反地，此值應該來自于目錄物件的 `commitTimestamp` 值。</span><span class="sxs-lookup"><span data-stu-id="ff249-448">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="ff249-449">每次用戶端想要處理封裝來源上的新事件時，它只需要針對認可時間戳記大於其儲存資料指標的所有目錄專案查詢目錄。</span><span class="sxs-lookup"><span data-stu-id="ff249-449">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="ff249-450">在用戶端成功處理所有新的類別目錄專案之後，它會將剛剛處理的類別目錄專案的最新認可時間戳記記錄為新的資料指標值。</span><span class="sxs-lookup"><span data-stu-id="ff249-450">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="ff249-451">使用這種方法，用戶端就一定不會錯過封裝來源上發生的任何套件事件。</span><span class="sxs-lookup"><span data-stu-id="ff249-451">Using this approach, the client can be sure to never miss any package events that occurred on the package source.</span></span>
<span data-ttu-id="ff249-452">此外，用戶端永遠不需要在資料指標記錄的認可時間戳記之前，重新處理舊的事件。</span><span class="sxs-lookup"><span data-stu-id="ff249-452">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="ff249-453">這項功能強大的資料指標概念用於許多 nuget.org 背景工作，可用來將 V3 API 本身保持在最新狀態。</span><span class="sxs-lookup"><span data-stu-id="ff249-453">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
