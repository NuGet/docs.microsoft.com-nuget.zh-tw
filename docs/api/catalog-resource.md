### <a name="overview"></a><span data-ttu-id="c54db-424">總覽</span><span class="sxs-lookup"><span data-stu-id="c54db-424">Overview</span></span>

<span data-ttu-id="c54db-425">本節說明一種用戶端的概念，雖然不一定是託管的通訊協定，但應該是任何實用的類別目錄的用戶端實作的一部分。</span><span class="sxs-lookup"><span data-stu-id="c54db-425">This section describes a client concept that, although is not necessarily mandated by the protocol, should be part of any practical catalog client implementation.</span></span>

<span data-ttu-id="c54db-426">因為目錄是依時間編制索引的僅附加的資料結構，所以應該儲存在用戶端**游標**最多哪個時間點的用戶端已在本機處理類別目錄項目。</span><span class="sxs-lookup"><span data-stu-id="c54db-426">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="c54db-427">請注意，永遠不會使用用戶端的電腦時鐘會產生此資料指標的值。</span><span class="sxs-lookup"><span data-stu-id="c54db-427">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="c54db-428">相反地，值應該來自於目錄物件的`commitTimestamp`值。</span><span class="sxs-lookup"><span data-stu-id="c54db-428">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="c54db-429">每次用戶端想要處理封裝來源上的新事件，它需要查詢類別目錄的認可時間戳記的所有目錄項目大於其預存的資料指標。</span><span class="sxs-lookup"><span data-stu-id="c54db-429">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="c54db-430">用戶端已順利處理所有新的類別目錄項目之後，它會記錄只處理做為其新的資料指標值的類別目錄項目的最新的認可時間戳記。</span><span class="sxs-lookup"><span data-stu-id="c54db-430">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="c54db-431">使用這個方法，用戶端可以確定永遠不會錯過任何發生的套件來源的封裝事件。</span><span class="sxs-lookup"><span data-stu-id="c54db-431">Using this approach, the client can be sure to never miss any package events that occurred on the package source.</span></span>
<span data-ttu-id="c54db-432">此外，用戶端永遠不會有重新處理資料指標的已錄製的認可時間戳記之前的舊事件。</span><span class="sxs-lookup"><span data-stu-id="c54db-432">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="c54db-433">資料指標的這個功能強大的概念用於許多 nuget.org 背景工作和用來保持 V3 API 本身。</span><span class="sxs-lookup"><span data-stu-id="c54db-433">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
